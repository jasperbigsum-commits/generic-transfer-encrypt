# Flutter Plugin FAQ

## 1. 为什么还需要 HTTPS？

这个插件做的是应用层传输加密，不是 TLS 替代。HTTPS 仍然负责：

- 服务端身份校验
- 通道机密性
- 通道级抗中间人

## 2. 为什么文件上传默认不加密？

这是和 Spring 端保持一致的设计：

- 大文件二次加密成本高
- 需要流式处理时复杂度更高
- 当前项目优先保证完整性校验和协议统一

## 3. 为什么上传接口还要公钥？

上传本身不使用公钥加密文件内容，但通常业务侧会复用同一个客户端实例去调其他加密接口。当前包构造时保留 `publicKey` 作为统一入参，减少多种客户端模型并存。

## 4. 服务端返回 400 时还能拿到解密后的错误体吗？

可以。如果响应仍然是仓库定义的加密 JSON 信封，客户端会先解密，再抛出 `TransferEncryptHttpException`，并把解密结果放进 `responseBody`。

## 5. Flutter Web 可以用吗？

可以，前提是：

- 目标服务允许浏览器跨域
- 你的 `dart_sm` 依赖在 Flutter Web 构建目标下可正常工作
- 公钥分发方式符合你的安全要求

## 6. 这个包支持 dio 吗？

当前实现基于 `package:http`。如果你项目统一使用 `dio`，建议再包一层网关适配，把 `TransferEncryptClient` 当作协议编解码器使用，或者继续扩展一个 `dio` 版适配器。
